## 💡 CONCEPTOS CRÍTICOS PARA MEMORIZAR

```
1. Big O Ranking
   1 < log n < n < n·log n < n² < 2ⁿ < n!

2. Búsqueda Binaria
   Precondición: ORDENADO
   Complejidad: O(log n)

3. Merge Sort
   Garantizado: O(n log n)
   Estable: SÍ
   
4. Quicksort
   Promedio: O(n log n)
   Peor: O(n²)
   
5. Recursión
   Necesita: caso base + reducción
   
6. Hash
   Colisiones: inevitables
   Factor de carga: n/m
```

## 📞 ATRAPADO EN UN PROBLEMA?

4. **¿Ordena datos?** → ¿Qué tamaño?
5. **¿Acceso rápido?** → Probablemente hash

## 📝 ESTRUCTURA DE RESPUESTAS

Si el examen te pregunta:

### "¿Complejidad de...?"
1. Identifica bucles
2. Determina anidamiento
3. Resuelve recurrencia si es recursiva
4. Simplifica notación
5. Da respuesta con justificación

### "¿Cuándo usar...?"
1. Lista precondiciones
2. Considera tamaño de datos
3. Considera número de operaciones
4. Compara con alternativas
5. Justifica tu elección

### "¿Por qué O(n²)?"
1. Identifica bucles anidados
2. Cuantifica el trabajo
3. Simplifica (n² término dominante)
4. Justifica por qué otros términos se ignoran

### "¿Mejora la solución?"
1. Identifica cuello de botella
2. Sugiere estructura mejor
3. Analiza complejidad nueva
4. Compara mejora
